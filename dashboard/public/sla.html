1 Overview

This Service Level Agreement (SLA) describes the performance and availability standards that QuantumGuard commits to for users of our quantum vulnerability scanning service. This SLA applies to the hosted version at quantumguard.site.

For self-hosted deployments, uptime is determined by your own infrastructure. See our Self-Hosting Guide for setup instructions.

5 Scheduled Maintenance

We perform scheduled maintenance during low-traffic periods to minimize disruption:

  • Scheduled maintenance window: Sundays 2:00 AM — 4:00 AM UTC
  • Users will be notified at least 24 hours in advance via GitHub
  • Emergency maintenance may occur without advance notice for critical security patches
  • Scheduled maintenance does not count toward downtime calculations

7 Data & Security Commitments

  • Source code is never stored — deleted immediately after scanning
  • All data transmitted over HTTPS / TLS 1.3
  • Database encrypted at rest using AES-256 (Google Cloud)
  • GitHub tokens never persisted — exist only in request memory
  • Rate limiting applied to prevent abuse and ensure fair access
  • Full codebase open source at github.com/cybersupe/quantumguard

8 Exclusions

The following are excluded from SLA calculations:

  • Scheduled maintenance windows
  • Free tier cold start delays (up to 50 seconds after inactivity)
  • Issues caused by GitHub API outages or rate limits
  • Force majeure events (natural disasters, internet outages)
  • Issues caused by user error or misuse of the API
  • Third-party service outages (Firebase, Vercel, Render)
