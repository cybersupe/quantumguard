3 How Your Code Is Handled

When you scan a repository, here is exactly what happens:

  • Your GitHub URL is sent to our backend over HTTPS
  • Backend downloads the repo as a ZIP file from GitHub
  • ZIP is extracted to a temporary folder (/tmp) on our server
  • Scanner reads the files and checks for vulnerability patterns
  • Results (score and findings) are sent back to you
  • The temporary folder is immediately deleted using shutil.rmtree() in a finally block — this runs even if an error occurs

Your source code never touches our database. It exists only in server memory for the duration of the scan request — typically less than 30 seconds.

4 Authentication & Third Parties

We use Google OAuth 2.0 for authentication. When you sign in with Google:

  • You authenticate directly with Google — we never see your password
  • Google sends us your name and email address only
  • Your session is managed using Firebase Authentication (Google Cloud)

We use the following third-party services:

  • Firebase / Google Cloud — Authentication and Firestore database. Subject to Google's Privacy Policy.
  • Vercel — Frontend hosting. Subject to Vercel's Privacy Policy.
  • Render.com — Backend hosting. Subject to Render's Privacy Policy.
  • EmailJS — Email delivery for scan reports. Subject to EmailJS Privacy Policy.

5 Data Retention & Deletion

Scan history records are retained as long as your account is active. You can delete individual scan records from your Scan History page at any time.

To delete your entire account and all associated data, email us at thisispayyavula@gmail.com with the subject "Delete My Account". We will process your request within 7 days.

6 Security

We take security seriously:

  • All traffic is encrypted using HTTPS / TLS 1.3
  • Database is encrypted at rest (Google Cloud AES-256)
  • Rate limiting prevents abuse and DDoS attacks
  • Our entire codebase is open source — you can audit our security claims at github.com/cybersupe/quantumguard
